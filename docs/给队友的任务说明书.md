# Growth OS · 给你的任务说明书(设计 / 产品 · 你)

> 队友,这是你在这个项目里要做的事。**代码全是 Zack 写,你一行都不用写。**

## 0. 你的角色(一句话)

你是 **AI 内容 + 体验 + 演示** 的负责人。Zack 把管道(代码)跑通,
你负责**喂好 AI、做好界面、讲好故事**。这三样在评委眼里,和代码同等重要。

## 1. 工作总原则(先记住)

1. **并行,不等他**:你永远有 Mock 数据 / Prompt / 设计 / 文案可做,不会卡住。
2. **产出他能直接用的格式**:文件命名、目录、格式都按本说明书第 6 节来,他拿过去就能用。
3. **完成一项就交接**:在共享文档勾掉并 @ 他,不要攒着。
4. **早 / 中 / 晚 各对齐 10 分钟**:说清进度 + 卡点,其余时间各干各的。

## 2. 你的 5 条任务主线(按重要性排序)

### 主线 ① Prompt 工程【最重要,决定产品质量】
- **做什么**:写好并调优这 4 个 Prompt,让 AI 输出真的能落地、跑得稳。
  1. Goal Decomposer(目标拆解)
  2. Daily Planner(每日任务生成)
  3. Behavior Analyst(行为分析)
  4. System Adjuster(动态调整)
- **给你的 Agent 指令模板(直接复制用)**:
  ```
  我在做成长类 App,需要你扮演【人生规划教练】。
  给定用户目标,输出:长期能力、3 阶段路径、可执行任务。
  约束:必须能落到【每日具体动作】。
  请用以下 3 个目标各跑一遍:"考研上岸" / "3 个月减 10kg" / "学英语到能开会",
  每个输出后自我点评"哪里不够落地",然后给改进后的【最终版 Prompt】。
  ```
- **交付物**:`prompts/01_goal_decomposer.txt` … 共 4 个纯文本文件;每个再配一份「示例输入 → 输出」对照。
- **验收**:同一个目标跑 3 次,输出结构一致、都能落到每日动作。
- **截止**:阶段 0–12h(Zack 12h 开始接线,你得在这之前交)。

### 主线 ② Mock 数据 / 种子数据
- **做什么**:让前端和演示**不等后端**就能跑。用 AI 生成各接口的假返回 JSON,外加一份真实感的演示数据。
- **给你的 Agent 指令模板**:
  ```
  按下面的数据模型,为 3 个典型用户各生成一套完整演示数据:
  【考研党 / 健身党 / 学英语党】,每人:1 个目标、对应 14 天的每日任务、
  对应行为日志(含中断和质量评分)、5 条系统记忆洞察。
  输出合法 JSON,字段严格按模型:
  User(id,name,baseline) / Goal(id,user_id,title,category,time_horizon,status)
  / Task(id,goal_id,date,content,difficulty,status)
  / BehaviorLog(id,task_id,done,quality,notes)
  / Memory(id,user_id,insight_type,content)
  ```
- **交付物**:`mock/` 目录:`goals.json`、`tasks.json`、`logs.json`、`memory.json`、`demo_users.json`。
- **验收**:JSON 合法、字段对得上模型、能直接喂给前端。
- **截止**:阶段 0–8h。

### 主线 ③ UI 设计 + 界面文案
- **做什么**:出 Dashboard / 打卡 / 时间视图 / 记忆面板 的信息架构和界面(Figma 或手绘截图都行),再加全套文案(含空状态)。
- **给你的 Agent 指令模板**:
  ```
  为一个"把人生拆成可计算系统"的成长 App 设计 Dashboard,强调系统感:
  今日完成率 / 连续天数 / 各维度(学习/健康/行为/目标)进度 / 趋势图 / 记忆面板。
  先给信息架构(每块区域放什么),再给每块的文案(空状态、按钮、提示),风格简洁专业。
  ```
- **交付物**:`design/` 目录:线框/截图 + `copy.md`(文案表)。
- **验收**:每个页面信息架构清晰、文案齐全(含空状态)。
- **截止**:阶段 4–24h。

### 主线 ④ Demo 剧本 + Pitch + README
- **做什么**:设计演示动线(用 3 个典型用户故事串成一条路)、写 Pitch、写 README。
- **给你的 Agent 指令模板**:
  ```
  帮我的黑客松项目 Growth OS 写 3 分钟演示脚本:用"考研党小张"从设目标
  到系统自动调整计划的完整故事串起来,突出"系统会反过来改计划"这个差异点。
  再写 500 字 Pitch:问题、方案、为什么这不是又一个打卡 App。
  ```
- **交付物**:`docs/demo_script.md`、`docs/pitch.md`、`README.md`。
- **验收**:故事流畅、卖点清晰、能脱稿讲。
- **截止**:阶段 24–44h。

### 主线 ⑤ 用户测试 + Bug 清单
- **做什么**:产品能跑后,你当第一个真实用户全程走一遍,记体验问题 + bug,按优先级排好交给他修。
- **给你的 Agent 指令模板**:
  ```
  这是我走查时的笔记:[贴笔记]。帮我整理成带优先级的清单
  (P0 阻塞 / P1 影响 / P2 优化),每条:现象、复现步骤、建议。
  ```
- **交付物**:`docs/bugs.md`(带优先级)。
- **验收**:可复现、有优先级、有建议。
- **截止**:阶段 34–44h。

## 4. 你永远不缺事做的「并行清单」
- 后端没好 → 你产 Mock,前端先跑。
- 界面没出 → 你写 Prompt、调质量、造种子数据。
- Prompt 调好了 → 你做设计、写文案。
- 都做完了 → 你写 Demo 剧本、Pitch、当用户测试。
> **原则:你手里永远有 3 件能独立推进的事,不靠他也能往前走。**
