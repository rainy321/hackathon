# Growth OS · 技术实现

本文档介绍 Growth OS 的核心技术结构,用于快速理解项目如何把长期目标、每日任务、行为记录和 AI 反馈串成闭环。

## 技术栈

- 前端:Next.js + React + Tailwind CSS
- 后端:FastAPI
- 数据库:SQLite
- AI:智谱 GLM(OpenAI 兼容协议)
- 架构原则:单 Agent + 4 个 Prompt 函数,不依赖多 Agent 框架

## 系统架构

```text
用户界面(Next.js)
    ↓ HTTP API
后端服务(FastAPI)
    ↓ 读写
SQLite 数据库
    ↘ 调用
GLM 大模型
```

后端负责统一调度:读取用户目标、任务、行为日志和长期记忆,把结构化上下文传给 AI,再将 AI 输出保存回数据库并返回给前端。

## AI 闭环

Growth OS 使用 4 个 Prompt 函数完成核心闭环:

1. **Goal Decomposer**:把长期目标拆成阶段计划和关键能力。
2. **Daily Planner**:根据当前阶段生成当天可执行任务。
3. **Behavior Analyst**:分析近期行为日志,识别完成率、阻碍和节奏偏差。
4. **System Adjuster**:根据分析结果调整任务难度、排期和执行策略。

所有 Prompt 都是纯文本文件,位于 `prompts/` 目录。调整 Prompt 不需要修改后端业务代码。

## 数据流

以生成今日任务为例:

1. 前端向后端发送目标或规划请求。
2. 后端从 SQLite 读取目标拆解、当前阶段、用户基础信息和历史行为。
3. 后端调用对应 Prompt 函数,请求 GLM 返回结构化 JSON。
4. 后端校验并保存任务、分析结果或记忆。
5. 前端刷新仪表盘、任务清单和 AI 反馈模块。

## 容错机制

- 没有配置 `GLM_API_KEY` 时,系统会自动使用 Mock 输出,本地仍可跑通完整闭环。
- AI 调用失败或返回格式异常时,后端会回退到结构化 Mock,避免核心流程中断。
- SQLite 初始化和种子数据灌入是幂等的,方便本地开发和轻量部署。
