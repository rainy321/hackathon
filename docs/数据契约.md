# Growth OS · 数据契约

> 这是 5 张核心数据表的字段说明。建库、种子数据、前后端接口和 Prompt 输入输出都以此为准。
> 当前契约已与项目种子数据和后端 schema 对齐。

## 字段约定

| 字段 | 约定 |
|---|---|
| 所有 `id` | **字符串主键**(如 `user_001`) |
| `difficulty` | 枚举字符串 **`低 / 中 / 高`** |
| task `status` | **`pending / completed / skipped`** |
| goal `category` | 中文 **`学习 / 健康`**(可扩展 `行为 / 目标`) |
| goal `status` | **`active / paused / done`** |
| `done` | 存 **INTEGER 0/1**;传给 Prompt 时转换为 boolean |
| `behavior_log.date` | 从关联任务的 `task.date` 带入 |
| `memory.content` | 保存长期记忆的自然语言内容 |

---

## 表结构

### user
| 字段 | 类型 | 说明 |
|---|---|---|
| id | TEXT PK | 如 `user_001` |
| name | TEXT | 用户名 |
| baseline | TEXT | 当前状态基线(自然语言) |

### goal
| 字段 | 类型 | 说明 |
|---|---|---|
| id | TEXT PK | 如 `goal_001` |
| user_id | TEXT → user.id | 所属用户 |
| title | TEXT | 目标标题 |
| category | TEXT | `学习 / 健康` |
| time_horizon | TEXT | 如 `3个月` |
| status | TEXT | `active / paused / done` |

### task
| 字段 | 类型 | 说明 |
|---|---|---|
| id | TEXT PK | 如 `task_k_001` |
| goal_id | TEXT → goal.id | 所属目标 |
| date | TEXT | `YYYY-MM-DD` |
| content | TEXT | 今天做什么 |
| difficulty | TEXT | `低 / 中 / 高` |
| status | TEXT | `pending / completed / skipped` |

### behavior_log
| 字段 | 类型 | 说明 |
|---|---|---|
| id | TEXT PK | 如 `log_k_001` |
| task_id | TEXT → task.id | 关联任务 |
| date | TEXT | `YYYY-MM-DD`,从 task.date 带入 |
| done | INTEGER | `0 / 1` |
| quality | INTEGER | `1-5` |
| notes | TEXT | 备注 / 中断原因 |

### memory
| 字段 | 类型 | 说明 |
|---|---|---|
| id | TEXT PK | 如 `mem_k_001` |
| user_id | TEXT → user.id | 所属用户 |
| insight_type | TEXT | 洞察类别(高效时段 / 障碍识别 / …) |
| content | TEXT | 洞察内容 |

---

## 4 个 Prompt 的输入/输出契约

| Prompt | 读 | 输出 JSON 结构 |
|---|---|---|
| 01 Goal Decomposer | `{goal, time_horizon, user_baseline}` | `{long_term_skills[], phases[{name,duration,tasks[]}]}` |
| 02 Daily Planner | `{goal_id, phase, yesterday_tasks[{content,done}], user_baseline}` | `{tasks[{content,difficulty(低中高),estimated_minutes}]}` |
| 03 Behavior Analyst | `{logs[{date,done,quality,notes}]}` | `{insights[{insight_type,content,confidence}]}` |
| 04 System Adjuster | `{completion_rate_7d, insights[], current_tasks[]}` | `{adjustments{tasks_to_reschedule[],difficulty_change,focus_strategy,reason}}` |

> 注意:Prompt 03 的输入 `logs` 需要包含 `date`,调用时从 `behavior_log.date` 读取。
