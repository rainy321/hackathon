# 路演 PPT · 技术实现(给队友讲)

> 目的:让评委一听就觉得"这是真做出来的、有技术含量",但不堆术语。每页有「讲什么 + 怎么说」。

---

## 🎞️ 幻灯片 1:技术栈(开门见山)

**标题:技术架构**

- 前端:**Next.js** · 后端:**FastAPI** · 数据库:**SQLite** · AI:**GLM 大模型(智谱)**
- 一句话定位:**「全栈轻量架构,核心是一个数据驱动的 AI 闭环」**

🎤 *怎么说*:"我们用主流全栈技术,后端接入智谱 GLM 大模型。重点不是堆技术,而是把它做成一个**会自我调整的闭环**。"

---

## 🎞️ 幻灯片 2:系统架构图(画进 PPT)

```
[ 用户 / 网页 Next.js ]
        ↕  发请求 / 显示结果
[ 后端 FastAPI ]  ← 调度中枢:取数据、调 AI、存结果、回传
   ↕ 读/写              ↘ 把「数据+指令」发给 AI
[ 数据库 SQLite ]        [ 大模型 GLM ]
 用户/目标/任务/           ↙ 返回结构化结果
 打卡/记忆(5 张表)
```

- **4 个角色**:网页(顾客)、后端(服务员+后厨)、数据库(账本)、AI(按指令干活的顾问)。

🎤 *怎么说*:"后端是个**调度中枢**——把用户的真实数据和精心写的指令打包交给 AI,再把结果存下来、返回给用户。"

---

## 🎞️ 幻灯片 3:核心实现路径(一次 AI 生成的 5 步)

**标题:它是怎么"生成"的** · 以"生成今日任务"为例:

1. 用户点按钮 → 前端发请求给后端
2. 后端**从数据库读**:目标、当前阶段、昨天完成情况
3. 后端**打包【数据 + 指令(Prompt)】→ 调用 GLM**
4. GLM **返回结构化结果**(今日任务,JSON)
5. 后端**存进数据库** + 返回前端显示

🎤 *怎么说*:"每一步都可追溯,**结果会落库**,所以 AI 不是随口一说,而是基于你的真实数据算出来的。"

---

## 🎞️ 幻灯片 4:AI 怎么接的(差异化,重点讲)

**标题:AI 工程设计**

- **单 Agent + 4 个指令函数**:目标拆解 / 每日规划 / 行为分析 / 动态调整
- **刻意不用**多 Agent 框架(LangChain / AutoGen)→ 更轻、更稳、指令可独立迭代
- **数据驱动**:建议基于真实打卡,不是模板话术
- **Mock 兜底**:没网/key 也能跑通闭环,演示零风险

🎤 *怎么说*:"我们**有意保持轻量**——不堆框架,把功夫花在**指令质量和闭环设计**上。这反而更稳、更可控。"

---

## 🎞️ 幻灯片 5:核心闭环 = 技术管道(收尾高光)

**设目标 → AI 拆解 → 每日任务 → 打卡 → AI 分析偏差 → 动态调整 → 写入记忆 → ↺ 循环**

- 每个箭头 = 一次"数据流经 后端→AI→数据库"
- 记忆会反哺下一轮 → **越用越懂你**

🎤 *怎么说(收尾)*:"这不只是个打卡工具,而是一个**会自我进化的成长系统**——它记住你的习惯,并反过来帮你调整计划。"

## 💬 一句话总结(可背下来收尾)

> "后端是一个**数据驱动的调度中枢**:它把用户的真实行为数据和精心设计的指令交给大模型,生成可执行的结果并记住,形成一个**会自我调整的成长闭环**。技术上我们有意保持轻量,把功夫花在 AI 指令和闭环设计上。"
